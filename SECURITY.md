# 安全策略

## 支持的版本

我们目前支持以下版本的安全更新：

| 版本 | 支持状态 |
| --- | --- |
| 1.0.x | ✅ |

## 报告漏洞

如果你发现了安全漏洞，请不要在公开的 Issue 中报告。

请通过以下方式私下报告：

1. 发送邮件到：yolaucn@gmail.com
2. 邮件标题：[SECURITY] 安全漏洞报告
3. 请包含以下信息：
   - 漏洞的详细描述
   - 重现步骤
   - 可能的影响
   - 建议的修复方案（如果有）

## 响应时间

- 我们会在 48 小时内确认收到你的报告
- 我们会在 7 天内提供初步评估
- 我们会在 30 天内发布修复（如果确认是漏洞）

## 安全最佳实践

使用这个项目时，请遵循以下安全最佳实践：

1. **保护 API 密钥**
   - 永远不要将 API 密钥提交到版本控制
   - 使用环境变量存储敏感信息
   - 定期轮换 API 密钥

2. **网络安全**
   - 在生产环境中使用 HTTPS
   - 验证所有外部输入
   - 实施适当的错误处理

3. **依赖管理**
   - 定期更新依赖
   - 使用 `npm audit` 检查已知漏洞
   - 只安装必要的依赖
